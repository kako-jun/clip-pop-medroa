# 今後の展望

コードの構造や設計上の決定から読み取れる、将来的な方向性をまとめます。

## カーソル追従表示

dev ノートに「マウスカーソル横にも色を出しては」というメモが残されています。
固定コーナーではなく、カーソルの近くにバナーが現れるモードの追加が検討されています。
「何をコピーしたか」をカーソルの文脈でその場で確認できる体験につながります。

## カスタム画像の充実

現在はPNG/WebP/GIFのファイルパス指定に対応しています。
アニメーション透過WebPのサポートはすでに組み込まれており、
より豊かな視覚表現（自作スタンプ、アニメーションキャラクター等）への拡張余地があります。

## 表示位置の拡張

現在は4隅への固定表示ですが、カーソル追従と組み合わせることで
「コピーした場所の近く」に通知を出すアプローチも自然な発展です。

## テーマの拡張

カスタムテーマは現在「画像を表示する」という形ですが、
色・フォント・角丸・透明度などをより細かく調整できる仕組みへの発展が考えられます。

## クリップボード内容のプレビュー

現在は「コピーした」という事実だけを通知します。
コピーされたテキストの冒頭を通知バナーに表示する機能は、
構造上（`ClipboardPayload` に `text` フィールドがすでにある）すぐに追加できる状態です。

## 自動起動

OS起動時に自動で常駐する設定はまだ実装されていません。
Tauri v2 のオートスタート機能を使えば対応可能です。

## タスクトレイ常駐

現在はバナーの「︙」メニューからしか操作できません。
システムトレイアイコンからの設定・終了操作を追加することで、
バナーが表示されていない状態でも操作できるようになります。
