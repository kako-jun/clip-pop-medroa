# プラットフォーム対応

## 対応OS

| OS | 対応状況 |
|---|---|
| Windows 10 / 11 | 対応 |
| macOS 12以降 | 対応 |
| Linux (X11 / Wayland) | 対応 |

## Windows

インストーラ形式（`.msi` または `.exe`）で配布されます。
`%APPDATA%\ClipPopMedroa\` に設定ファイルが保存されます。
常時最前面の透明ウィンドウは Windows のウィンドウ管理と協調して動作します。

## macOS

`.dmg` 形式で配布されます。
`~/Library/Application Support/ClipPopMedroa/` に設定ファイルが保存されます。
macOS のセキュリティポリシーにより、初回起動時にクリップボードへのアクセス許可を求められる場合があります。

## Linux

`.AppImage` および `.deb` 形式で配布されます。
XDG規格に従い、`~/.config/ClipPopMedroa/` に設定ファイルが保存されます（`$XDG_CONFIG_HOME` が設定されている場合はそちらを優先します）。
X11 および Wayland の両環境で動作します。ただし、GUIのテストにはディスプレイ環境が必要です。

## クリップボード監視の仕組み

クロスプラットフォームの互換性を確保するため、OS固有のクリップボードフックではなく、ポーリング方式を採用しています。
約900ミリ秒ごとにクリップボードの内容を確認し、前回からの変化を検出します。
この方式はすべての主要プラットフォームで同じ動作を保証します。

## 既知の制限

- クリップボードの変化検出はテキストのみを対象としています。画像や他の形式のデータはイベントとして検出されません
- ポーリング間隔（約900ミリ秒）以内に起きた複数の変化は、最後の状態のみが通知されます
